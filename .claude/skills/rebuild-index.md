# Rebuild Index

Regenerate `INDEX.md` at the repo root with a comprehensive, categorized document index.

## Why This Exists

INDEX.md is the master catalog of all documents in PersonalOS. It enables fast lookups without grep and gives Claude a single file to read for "what exists where." It must stay current to be useful.

## What Gets Indexed

Scan these directories (in order):

1. **Tasks/*/** -- all task files and subfolder contents (skip archive/)
2. **Tasks/*.md** -- root-level task files (TODO.md, DASHBOARD.md, etc.)
3. **Knowledge/meetings/YYYY-MM/** -- meeting notes by month
4. **Knowledge/meetings/recurring/*/META.md** -- recurring series metadata
5. **Knowledge/decision-log.md** -- decision log (if it exists)
6. **Knowledge/misc/** -- reference docs
7. **Documents/** -- personal/career docs (skip archive/)
8. **GOALS.md, BACKLOG.md** -- root-level system files

Skip: `core/`, `examples/`, `.claude/`, `.github/`, `node_modules/`, `sensitive/`

## Index Format

```markdown
# Index

> Auto-generated by `/rebuild-index` on YYYY-MM-DD HH:MM. Do not edit manually.

## Quick Stats
- **Tasks:** X active (Y with subfolders)
- **Meetings:** X notes across Y months + Z recurring series
- **Documents:** X files

---

## Tasks (Active)

| Task | Priority | Status | Category | Location |
|------|----------|--------|----------|----------|
| Task title | P0 | s | technical | Tasks/Q1/task-name/ |

## Tasks (Root)

| File | Description |
|------|-------------|
| [TODO.md](Tasks/TODO.md) | Master to-do list |

## Meetings by Month

### 2026-04
| Date | Meeting | Participants |
|------|---------|--------------|
| 2026-04-07 | Weekly sync | Alice, Bob |

### 2026-03
...

## Recurring Series

| Series | Cadence | Participants | Active Topics |
|--------|---------|--------------|---------------|
| Team standup | Daily | Full team | 5 |

## Knowledge & Reference

| File | Category | Description |
|------|----------|-------------|
| decision-log.md | Decisions | Project decisions source of truth |

## Documents

| File | Description |
|------|-------------|
| personal-os-narrative.md | System overview |

---
*Last rebuilt: YYYY-MM-DD HH:MM*
```

## Steps

### Step 1: Scan task files

1. Use Glob to find all `.md` files in task quarter folders (both flat files and `task.md` inside subfolders)
2. Read YAML frontmatter from each to extract: title, priority, status, category
3. Skip `status: d` tasks (they should be in archive)
4. Count tasks with subfolders vs flat files

### Step 2: Scan meetings

1. Use Glob to find all `.md` files in `Knowledge/meetings/YYYY-MM/` directories
2. Extract date and title from filename (no need to read full file)
3. Read each recurring series META.md to get cadence, participants, active topic count
4. Group by month, sort newest first

### Step 3: Scan Knowledge and Documents

1. Glob for files in `Knowledge/misc/`, `Knowledge/career/`, `Knowledge/emails/`
2. Glob for files in `Documents/` (skip `Documents/archive/`)
3. Read first few lines of each for a brief description if available

### Step 4: Build quick stats

Count totals from each section.

### Step 5: Write INDEX.md

1. Assemble the full index using the format above
2. Write to `INDEX.md` at repo root
3. Include timestamp in the footer

### Step 6: Report

Show the user:
- Total documents indexed
- Any issues found (orphaned files, missing frontmatter, etc.)
- If `$ARGUMENTS` is "full", show the complete index; otherwise just the stats summary

## Rules

- Do NOT read full file contents for meetings (too many files). Extract info from filenames and frontmatter only.
- Keep the index under 300 lines. If meetings exceed this, show only last 3 months and add a "N older months in archive" note.
- For task descriptions, use the YAML `title` field, not the filename.
- Always overwrite the existing INDEX.md completely.
